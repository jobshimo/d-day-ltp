# TTS Narration Tooling

Pre-generates MP3 narration for every prose and rule-callout lesson block using
[Kokoro TTS](https://github.com/hexgrad/kokoro) running **locally** — no network,
no accounts required.

Voice: `em_alex` (Spanish male). Language code: `e`.

---

## Prerequisites

The Python virtual environment is **not committed** (it is excluded by `.gitignore`).
Set it up once:

```bash
# From the repo root
cd tools/tts
python -m venv .venv
.venv\Scripts\activate          # Windows
# source .venv/bin/activate     # Linux / macOS

pip install -r requirements.txt
```

`requirements.txt` installs Kokoro, espeakng-loader (bundles espeak-ng, no system
dependency), soundfile, and numpy.

---

## Two-step generation pipeline

### Step 1 — extract the manifest

```bash
# From repo root  (Node 22+)
node --experimental-strip-types tools/tts/extract-narration.mjs
```

Fall-back if type-stripping chokes on imports:

```bash
npx -y tsx tools/tts/extract-narration.mjs
```

This reads `libs/content/src/index.ts` → `ALL_MODULES`, collects every narratable
block (type `prose` or `rule-callout`), and writes:

- `tools/tts/narration-manifest.json` — source of truth consumed by the Python script
- `libs/content/src/narration.generated.ts` — committed TS module with `NARRATION`
  map and `getBlockNarration()` helper

### Step 2 — synthesize audio

```bash
# From repo root
PYTHONUTF8=1 tools/tts/.venv/Scripts/python tools/tts/generate.py
```

On Linux / macOS:

```bash
PYTHONUTF8=1 tools/tts/.venv/bin/python tools/tts/generate.py
```

Reads the manifest and writes `libs/content/src/assets/audio/<hash>.mp3`.
Files are **content-addressed** (SHA-1 of `voice + "\n" + text`, first 12 hex chars),
so they are never re-synthesized when the text has not changed.

---

## When to regenerate

Re-run the two-step pipeline any time lesson prose changes.  Content-addressed
filenames guarantee automatic deduplication: unchanged blocks are skipped, and
the old MP3 (with a different name) becomes unreferenced and can be removed
by hand when desired.

---

## Important notes

- **Always feed properly accented UTF-8 text** (é, í, ó, ú, ñ…). Accent-stripped
  input causes Kokoro to mispronounce Spanish words (e.g. "infantéria" instead of
  "infantería"). The real lesson content in `libs/content/` is correctly accented.
- The Python script must be run with `PYTHONUTF8=1` so Python opens files in UTF-8
  mode on Windows.
- espeak-ng is **bundled** via `espeakng-loader` — no system-level espeak package
  is required.
- German proper nouns (Widerstandsnest, Kampfgruppe, etc.) are phonetized with
  Spanish phonetics under `lang_code='e'`. This is acceptable for v1; a
  pronunciation dictionary can be added in future iterations.

## Files

| File | Role |
|------|------|
| `narration.config.json` | Single source of truth for voice / lang / sampleRate |
| `extract-narration.mjs` | Node ESM script — reads content, writes manifest + TS |
| `generate.py` | Python script — reads manifest, calls Kokoro, writes MP3s |
| `requirements.txt` | Python dependencies for the `.venv` |
| `.venv/` | Local Python venv (gitignored) |
| `narration-manifest.json` | Generated by step 1; consumed by step 2 |

Generated artifacts that **are committed**:

- `libs/content/src/narration.generated.ts`
- `libs/content/src/assets/audio/*.mp3`
